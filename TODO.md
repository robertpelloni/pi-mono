# Crucial Code Review Fixes (Next Session)
1. **Fix `StopReason` Mapping in Go Streams:** The Go HTTP streams currently default to `StopReasonStop` on loop termination. However, if the provider stream indicates `finish_reason: "tool_calls"` (OpenAI) or `stop_reason: "tool_use"` (Anthropic), the parser should explicitly set the stop reason to `ai.StopReasonToolUse`. Update `pkg/ai/openai.go`, `anthropic.go`, and `google.go` to parse and assign this correctly to the final event.
2. **Move to Select Context Cancellation:** While the `1000` size buffer successfully prevents the goroutine leak on context cancel, the code review noted that explicitly wrapping the stream sends in `select { case <-reqCtx.Done(): return; case stream <- event: }` is mathematically safer for indefinitely large generation outputs. Refactor the stream channels back to unbuffered and apply this `select` pattern across all providers.
